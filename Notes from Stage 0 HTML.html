NOTES for HTML:

Definition: HTML makes the structure of a website.

Every HTML file should start with "<!DOCTYPE html>" and wrap all the code within "<html>"
Not closing a tag will change the rest of the text to the format that was not closed. Pay attention to opening and closing tags.
Structure of an HTML: 
					
						<!DOCTYPE HTML>
						<html>
						<header>
							<title> SOME TITLE </title>
							<link rel="stylesheet" href="style.css">
						</header>
						<body>
							<h1>
								<p>
									SOME PARAGRAPH
								</p>
						</body>
						</html>
Everything is a BOX, no matter the shape of the image:
  • Boxes help organize the structure of the website easily and rearrange them if needed.
Front-End Web development: shows the result of the code instantly.
Structure: HTML code usually follow a tree-like structure. As seen before in the example structure of HTML; code is written within a bigger structure that goes to a smaller element, or better called a parent element where they have several child elements.
Indentation: even though not necessary for a program to work and is ignored by a computer; indentation has to be one of the most important elements in writing and reading code. As explained before in how structure works, indentation helps mark the different child elements that a parent has, by making it easier to view the different elements and how big a parent element can be.
Repetition: do NOT repeat code if it can be avoided! Repetition can lead to mistakes. The more code I write, the higher the probability of making a mistake or mistyping something that will make the program or website to crash. Use ID's, classes or other tools to avoid repetition and apply the same formats to different groups of texts.
Use Google Chrome for Developer tools to understand the inner structure of a website.
"<Style>" can be used to edit elements directly in HTML but is not considered good practice.

NOTES for CSS:

Definition: configures the style of the website by changing the color, fonts, etc. of the different objects that are presented in the HTML.

Adding style to HTML, there are 3 ways:
  • Linking a separate CSS file to the HTML: 
						
							<head>
								<link> rel="stylesheet" type="text/css" href="style.css"
							</head>
    Make sure that the files are in the same folder, otherwise it won't work or specify where the file is located.
  • Writing the CSS in the HTML through "<style>"
  • Writing the CSS inline with the HTML code is considered bad practice and leads to repeated coding. Not repeating code will allow us to write less code that can be less confusing.
Font-Style: modifies ONLY to normal, italics, oblique (not sure what the difference between oblique and italics is?)
Font-Weight: modifies ONLY to normal, bold, lighter, bolder, and numeric values of boldness.
Classes: to call a class in CSS, the element should be opened with ".". They can call a group of elements.
ID: to call an ID in CSS, the element should be opened with"#". ID is an unique identifier of a HTML element.
Selectors: are the component that you use to select or target a specific element in a HTML document.
Box-sizing: keeps the size of the boxes: 
						
							<-webkit-box-sizing: border-box;>
							<-moz-box-sizing: border-box;>
							<-ms-box-sizing: border-box;>
							<box-sizing: border-box;>
Set sizes to percentages rather than pixels since they can adjust to the screen size rather than a set of specific numbers.
Set the box-size = border-box.
Setting the display to "display: flex" will make the boxes of a DIV to appear next to each other horizontally rather than having everything lined up in a vertical way.

Notes for Computer Programming:

Definition: computers by itself cannot do much, unless a program is introduced and tells the computer what to do. A program uses the different components that a computer has available to perform the different commands that a program gives.

Notes for Python:

Definition: a programming language that enables computers to do specific commands in few lines of code. The main reason computer languages are created is because they serve a specific purpose and have specific commands; whereas normal languages (English, Spanish, Danish, etc.) can contain certain ambiguity that might confuse the computer or send a command that was not desired.

Python Interpreter: converts the code written by the programmer so that the computer can execute the commands.
Python Expression: a correct or legal Python statement that the interpreter is able to understand and execute. Incorrect expressions will lead to syntax errors and the program to crash.
Print: will display what we wrote in the command line in the console after we run the program.
Grammar: has to be very specific. If code is written that does not follow the python grammar, then it give us a syntax error.
Parameters: words or letters that go inside parentheses in a procedure and method. Within parameters you find inputs, operands or arguments.
General Solutions: code that can be applied to different situation no matter the whether the content changes inside the string.
Specific Solutions: code that only applies to the text written. Changes of a variable or string will affect the code and most likely changes will have to be applied.
Abstraction: the function of a computer when we call a specific module, function, or any other precedure or element without the coder actually specifying what it is being called.
Classes: contain several modules within and makes it easy to call different modules instead of writing several lines of code (eg: cookbook with receipes. The recipes is a class, since through a recipe different things can be done, but there is a standard followed).
Built-In Functions: a function that is commonly used and does not need to be imported. For this reason, we can write the function without the import portion.
Import: as the name suggests, it calls in a specific function from the python library. Commonly, functions that need to be imported are not typically used in a daily basis.
Modules: is a collection of different variables or functions that makes it easier to use Python, since with a few letters we can ask the computer to perform a specific task, instead of writing a whole procedure. There are different modules:
  • Turtle: is a graphic module that can be used to draw different elements on a computer screen.
    • window: creates a window where the graphic can be seen.
    • bgcolor: changes the color of the background in the window.
    • turtle.Turtle(): calling the function turtle, so the graphic is created.
    • str.shape: edits the shape of the avatar. The default is just and arrow.
    • str.color: modifies the color of the avatar.
    • str.speed: modifies how fast the avatar will move in the window.
    • str.forward(): how far the avatar will go in the window.
    • str.rigth/left(): direction in which the avatar will go.
  • OS Modules:
    • os.listdir(): marks where a certain folder or file is. However, is does NOT change the directory that python is working at. The code for listdir() is:
								
									os.listdir("path of file")
    • os.getcwd(): marks the current directory or folder in which Python is working at. The code for getcwd() is:
								
									os.getcwd()
    • os.chdir(): changes the directory or folder in which Python is working at after being specifying a new directory. The code for chdir() is:
								
									os.listdir("path of new directory")
    • str.translate(): removes some character of the names of the file/s that are found in the directory that Python is working at (in Python 3 writing "None" will not work anymore since it uses Unicodes). The code for str.translate() is:
								
									str.translate(None, "characters that we do not want to include")
    • os.rename(src, dst): changes the name of the file/s that are found in the cirectory that Python is working at. The code for os.rename() is:
								
									os.rename(src, dst)
Variables: stored location associated with a specific name. They also can vary on value.
  • Variables are composed of: name = expression
  • The equal sign "=" means assignment.
  • Variables can be multiplied and concatenated but CANNOT be concatenated with strings and variables in the same line of code.
Structured Data: can be divided into two different categories:
  • Strings: a way of storing a sequence of characters between quotes "" ''. In order for a string to work, it has to start and end with the same quote. Also, another string can be used in another string "'" '"'.
    • Indexing String: strings can be selected by a specific letter or expression through the [#]. ALWAYS start from 0 since computers start counting from that number!
    • Concatenation: the linking of 2 or more strings, but it CANNOT be multiplied.
    • Sub-Sequences: strings can also be selected by a group of letter or words by using [#:#]. ALWAYS start from 0, if trying to select the first word!
    • Finding a String: specific strings can be found through the method .find(string). If the string that is been searched is found, then it will give the number location within the entire string. In the case that .find does NOT find the string searched, it will give an output of -1. It can also have 2 parameters .find(#, #), where the second parameter specifies the program to start looking from the specified location. This can omit certain letters from a string or words.
    • Edge Case: an empty string where there are no characters and the interpreter returns an error.
    • Append: adds another element
  • List: a sequence that can store anything, including characters, integers, even another lists (also known as Nested Lists).
    • List Operations:
      • Append: adds another sequence to the list mutating the overall list.
    							
							 <variable/string>.append(<statement>)
      • +: concatonates the list with 2 or more lists.
    Procedures/Functions: a method that takes inputs as the parameters and outputs the different results that have been inputed within the parameters. The syntax for a function is: 
    							
								def <procedure>(<parameters>):
									<statements>
									return <statements>
    • Inputs: are arguments that are introduced to a method or procedure in order for it to perform a specif task.
    • Outputs: are the results of the different inputs that were introduced to method of procedure.
    • Return: will give us an output, otherwise the function or procedure wont display anything. This does NOT mean that the function does not work.
    • Making a Function: can be considered the section where the inputs will go and analyze the data.
    • Using a Function: also called calling a function, where the procedure is initialed and an output is expected.
    • Repetition: functions can be used several times and give different outputs depending on the inputs. However, if the structure of the function is not changed.
    • If-Else: a type of procedure or function that tests the expressions inputed and finds out whether an input is true or false, and can also take more than 2 inputs. the structure of an If-Else procedure is:
    							
							def <procedure>(<parameters>):
								if <statements>:
									return <statements>
								else <statements>:
									return <statements>
    • .find: is a method to locate a specific word or number whitin a string. The structure of find is:
    							
								<statement>.find[#:#]
    • Indentation in Procedures: have to be followed in specific orders or the interpreter will return an error.
    • Len: gives the number of elements found in the input. It also counts the outer elements. The syntax for len is:
    							
								len(<list>)
    Operators: offer different inputs to manipulate the prcedure in different ways.
    • OR: if either one of the statemenst proves to be true, the function will run; but if both of them prove false, the function will not run.
    • AND: in order for the procedure to run, both statements have to be true.
    • >: if a statement is bigger than another.
    • <: if a statement is smaller than another.
    • ==: if both statements are equal to each other.
    • !>: if both statements are NOT equal to each other.
    • >=; if a statement is bigger AND/OR equal to another.
    Loops: a method that allows us to do something over and over again as long as the text expression is true. If the test expression is false, the loop will stop.
    • While: a type of procedure that executes until the conditions is false. The format of a while loop is:
    							
								while <test expression>:
									<statements>
    • Break: a way to stop a loop after a statement is achieved. The syntax for break is:
    							
								def <procedure>(<parameters>):
									while <test expression>:
										<statements>
									if <statements>:
										return <statements>
										break
									else <statements>:
										return <statements>
    • For: a loop that goes throught the element and returns what if stated in the block. For loops do not run on an infinite way.
    • Indexing: there are 3 ways to index a look for a specific element:
      • If <value> is found in the <list>, then it returns the first position of the <value>; otherwise it returns an error. The syntax of this index is: 
    							
								<list>.index(<value>)
      • If <value> is found in the <list>, output is True; otherwise it is False. The syntax of this index is:
    							
								<value> in <list>
      • If <value> is NOT found in the <list>, output is True; otherwise it is False. The syntax of this index is:
    							
								<value> not in <list>
    • Infinite Loop: a loop that will never stop working because a test condition is not achieved.
    Debugging: an important part of coding is recognizing the mistakes and understanding the messages that the console gives you. However, some times the console is not too clear when it highlights that something went wrong.There are many approaches to find out what went wrong, such as editing small portions of the code to find out specifically where it crashed, comment portions of the code so the entire code does not run at the same time, include print statements so we can figure out if the code is doing what we ask for it to do.

    NOTES for DOM (Document Object Model):

    Definition: a cross-platform and language that interacts with objects from a HTML, XHTML, XML documents.

    • They are organized in a tree-like structure like HTML.

    Inline Vs. Block:

    Inline: Elements that take over one line.
    Blocks: Elements that are invisibly encircled
    • "<em>" Italics is an inline.
    • "<b>" Bold is an inline.
    • "<img src="" alt="">" Images is an inline.
    • "<br>" Break is an inline.
    • "<p>" Paragraph is a block.
    • "<span>" Span is an inline. Keeps everything within a line with no format.
    • "<div>" Div is a block. Good to group multiple elements.

    Div Vs. Span:

    Div: defines a section or division in a HTML. It is a block element! They also take the entire width of the web page.
    Span: provides a way to add a hook to a part of a text or a part of a document. It is a inline element!
